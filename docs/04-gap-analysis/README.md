# 04 - Gap Analysis

This phase compares the requirements with available tools, libraries, APIs, and project constraints.

The goal is to identify:

- what can be reused
- what must be built
- what is risky or unknown
- what should be validated with small prototypes before full implementation

## Deliverables

- `gap_analysis.md` - requirement-by-requirement gap analysis
- `build_vs_buy.md` - what to build, use, or postpone
- `risk_register.md` - project risks and mitigations
- `spike_candidates.md` - focused prototypes to run before implementation
