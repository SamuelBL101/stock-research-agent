# 02 - Use Case Analysis

This phase defines concrete scenarios:

- who uses the system
- what they are trying to do
- when the system acts
- what input the system needs
- what output the user expects
- what can go wrong

The output of this phase becomes the input for requirements.

## Deliverables

- `use_cases.md` - concrete use cases and flows
- `actors.md` - users and system actors
- `mvp_boundary.md` - what belongs in MVP and what waits
