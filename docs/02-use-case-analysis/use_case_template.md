# Use Case Template

Use this structure when adding new use cases.

## UC-XX: Name

### Goal

What the actor wants to accomplish.

### Actor

Who initiates or benefits from the use case.

### Trigger

When the use case starts.

### Preconditions

What must already be true.

### Main Flow

1. First step.
2. Second step.
3. Final step.

### Expected Output

- Output item

### Failure Cases

- Failure case

### Acceptance Criteria

- Testable result

### MVP Priority

High, medium, low, or future.
